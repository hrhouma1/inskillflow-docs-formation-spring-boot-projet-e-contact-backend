# Annexe A - Glossaire des termes

## A

**API (Application Programming Interface)**
Interface permettant a deux applications de communiquer entre elles.

**Annotation**
Metadata ajoutee au code Java avec @ (ex: @Service, @Entity).

**Authentification**
Processus de verification de l'identite d'un utilisateur.

**Autorisation**
Processus de verification des permissions d'un utilisateur.

## B

**Bean**
Objet gere par le conteneur Spring.

**BCrypt**
Algorithme de hachage securise pour les mots de passe.

**Builder Pattern**
Pattern de conception pour construire des objets complexes.

## C

**CORS (Cross-Origin Resource Sharing)**
Mecanisme permettant les requetes entre domaines differents.

**CRUD**
Create, Read, Update, Delete - operations de base sur les donnees.

**Controller**
Composant qui recoit les requetes HTTP et retourne les reponses.

## D

**DTO (Data Transfer Object)**
Objet pour transferer des donnees entre couches.

**Dependency Injection**
Technique ou les dependances sont injectees plutot que creees.

**Docker**
Plateforme de conteneurisation d'applications.

## E

**Entity**
Classe Java mappee a une table de base de donnees.

**Endpoint**
URL accessible d'une API.

## F

**Filter**
Composant qui intercepte les requetes HTTP.

## G

**GET**
Methode HTTP pour lire des donnees.

## H

**HTTP (HyperText Transfer Protocol)**
Protocole de communication web.

**Hibernate**
Implementation ORM de JPA.

## I

**IoC (Inversion of Control)**
Principe ou le framework controle le flux d'execution.

## J

**JPA (Java Persistence API)**
Specification pour le mapping objet-relationnel.

**JSON (JavaScript Object Notation)**
Format de donnees leger et lisible.

**JWT (JSON Web Token)**
Standard pour les tokens d'authentification.

## L

**Lombok**
Bibliotheque qui reduit le code boilerplate Java.

## M

**Maven**
Outil de gestion de projet et de dependances Java.

**MVC (Model-View-Controller)**
Pattern d'architecture separant donnees, presentation et logique.

## O

**ORM (Object-Relational Mapping)**
Technique de conversion entre objets et base relationnelle.

**OpenAPI**
Specification pour documenter les APIs REST.

## P

**POST**
Methode HTTP pour creer des donnees.

**PUT**
Methode HTTP pour remplacer des donnees.

**PATCH**
Methode HTTP pour modifier partiellement des donnees.

**Pagination**
Technique pour diviser les resultats en pages.

## R

**REST (REpresentational State Transfer)**
Style d'architecture pour les APIs web.

**Repository**
Composant d'acces aux donnees.

**RuntimeException**
Exception non verifiee (unchecked).

## S

**Service**
Composant contenant la logique metier.

**SMTP (Simple Mail Transfer Protocol)**
Protocole pour l'envoi d'emails.

**Spring Boot**
Framework Java pour creer des applications Spring.

**Spring Security**
Framework de securite pour Spring.

**Stateless**
Sans etat - chaque requete est independante.

**Swagger**
Outils pour documenter et tester les APIs.

## T

**Token**
Jeton d'authentification.

**Transaction**
Ensemble d'operations atomiques sur la base de donnees.

## U

**URI (Uniform Resource Identifier)**
Identifiant d'une ressource.

**UserDetails**
Interface Spring Security representant un utilisateur.

## V

**Validation**
Verification de la conformite des donnees.

**Volume (Docker)**
Mecanisme de persistance des donnees Docker.

## Y

**YAML (YAML Ain't Markup Language)**
Format de configuration lisible.
